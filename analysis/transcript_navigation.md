# Transcript Navigation Guide

Meeting transcripts typically follow predictable conversational patterns.

Understanding these patterns helps identify where important evidence is likely to appear.

---

# Typical Meeting Structure

Most meetings follow this structure:

1. Opening / greetings
2. Status updates
3. Feature demonstrations
4. Technical explanations
5. Timeline or planning discussion
6. Closing remarks

Evidence is most often found in sections 3, 4, and 5.

---

# Evidence Hotspots

Agents should focus attention on the following parts of transcripts.

## Feature Demonstrations

Often include:

- client reactions
- explanations of system behavior
- discussions of how the system works

These sections frequently contain evidence related to:

Client Sentiment  
System Behavior  
Technical Limitations

---

## Technical Explanations

These sections often contain the most important evidence.

Look for statements explaining:

- operating system behavior
- background processes
- location permissions
- system limitations

These sections frequently support:

Rehydration  
Location Permissions

---

## Timeline Discussions

These sections typically include:

- launch planning
- deadlines
- scheduling decisions

These statements support:

Project Scope  
Development Phasing

---

# Indicators of Important Statements

Agents should pay attention when speakers say phrases like:

"the reason this works like that is"

"iOS requires"

"we can't control"

"the operating system"

"the deadline was"

"we're trying to go live"

These often precede important technical or project decisions.

---

# Mining Strategy

Efficient transcript mining involves:

1. scanning for key phrases
2. identifying relevant sections
3. reading surrounding context
4. extracting high-quality quotes
