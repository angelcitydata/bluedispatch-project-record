# LES Mobile App Evidence Investigation

This repository contains transcripts, analysis materials, and extracted evidence related to the development of a mobile application for Law Enforcement Systems (LES).

The project involved weekly meetings between the development team and the client organization from May through December 2025.

The purpose of this repository is to construct a **verifiable Evidence Index** documenting statements made during those meetings regarding:

- technical limitations of the mobile platform
- client understanding of those limitations
- project scope and timeline expectations
- communication patterns
- client sentiment toward the project

The Evidence Index contains **timestamped quotes tied directly to transcripts**.

Each entry references the exact transcript file and timestamp so that every statement can be independently verified.

---

# Repository Structure

- /source
  Contains raw transcripts of weekly project meetings.

- /analysis
  Working documents used to navigate and analyze transcripts.

- /evidence
  Structured evidence entries extracted from transcripts.

- /reports
  High-level summaries and narrative findings.

---

# Areas of Concern (AoCs)

Evidence is organized into five investigative categories.

1. Rehydration / Force-Close / Session Recovery
2. Location Services / Permission Education
3. Project Scope / Development Phasing
4. Client Feedback / Project Sentiment
5. Project Communication / Meeting Record

Each evidence entry is categorized under one of these areas.

---

# Evidence Format

Each evidence entry includes:

- Date
- Meeting
- Source Transcript File
- Speaker
- Timestamp
- Verbatim Quote
- Significance Explanation

All evidence must remain **fact-based and traceable to the transcript source**.

Interpretation or conclusions belong in `/reports`.
